# Project Library (canonical swap inventory)

This is the **single source of truth** for project content. The resume firewall (`resumes.md`) swaps **from here**; the PM / Software / Physical resume masters are *generated views* of this library. **Never state a project metric from memory — read this file.**

Every entry is sourced from the applicant's final documents, resumes, and direct confirmation during the dossier-intake interview. Each ⚠️ flag is a do-not-overclaim guardrail; each ✅ marks a verified correction made against a prior overclaim.

---

## Anti-Fabrication Firewall

These rules govern every entry in this file. They are methodology, not preference — never skip them.

1. **Every number must be defensible.** If the applicant can't say "I could cite that in an interview and back it up," the number doesn't go in. Use `(estimated)` for approximations and flag them with ⚠️.
2. **Shipped ≠ concept ≠ roadmap.** Mark every entry's status clearly: shipped/live, shipped in controlled context (internal/pilot), academic/concept, or roadmap. Never let "we planned to" read as "we did."
3. **Titles print exactly.** No inflation. An intern is an intern. A contractor is a contractor. The honest title is a fact; an inflated title is a liability.
4. **Funding amounts = tranche, not total.** If only the first milestone of a funding round closed, state the tranche. "$1.2M SAFE — initial tranche" is honest. "$1.2M raised" when only $400K closed is overclaiming.
5. **Team credit is explicit.** "Led a team of 4" and "I single-handedly" are both honest in different situations. When the applicant did a subset of the work, say so.
6. **Regulated claims require proof.** Medical, accessibility (WCAG audits), safety, and security claims require explicit evidence of credential or methodology. Tag as `(working)` or `(learning)` unless proven otherwise — see `competencies.md` for the guardrail.
7. **Missing ≠ false.** A skill or metric not yet in this file is "unconfirmed," not absent. The right response is to ask the applicant, not to invent or to drop it silently.

## Entry Schema

```
### [Project / Role — Org] (Dates)
- **Type:** Role | Founder venture | Project | Shipped product | System
- **Domains:** [PM] / [SW] / [PH]   (which resume tracks it serves)
- **Verified metrics / facts:** numbers exactly as confirmed; ⚠️ flags inline
- **Bullet variants:** XYZ-form, outcome-frontloaded; [domain]-weighted where noted
- **ATS keywords carried:** comma-separated, only terms genuinely true of this work
- **Accuracy notes:** do-not-overclaim flags specific to this entry
```

**`Type` → resume section:** **Role** and **Founder venture** are positions held → the **Experience** section (reverse-chronological). **Project**, **Shipped product**, and **System** are bodies of work → the **Selected Projects** section (curated per JD), or as bullets inside the role that contained them. A founder venture is dual-natured (a role that also contains projects) — list it in Experience, and mine its internal projects for bullets.

## Worked Example (FABRICATED — replace entirely with real entries during intake)

### Acme Corp — Senior Widget Designer (Jan 2019 – Mar 2021) · Chicago, IL
- **Type:** Role
- **Domains:** [SW] primary · [PM] secondary
- **Verified metrics / facts:** Led a redesign of Acme's core widget-configuration flow; reduced average task completion time from 4.2 min to 1.8 min (measured via moderated usability study, n=24); shipped to ~80,000 monthly active users. Ran a mixed-methods research sprint (diary studies n=12 + intercept interviews n=30) to identify the top 5 drop-off points. Collaborated cross-functionally with a 3-person eng team and 1 PM over 6 months. ⚠️ The "80,000 MAU" figure is from the PM's Slack message — treat as approximate until verified from analytics dashboard.
- **Bullet variants:**
  - [SW] "Redesigned Acme's widget-configuration flow — cut average task completion time from 4.2 to 1.8 minutes (n=24 usability study) — shipping to ~80,000 monthly active users."
  - [PM] "Led discovery-to-delivery for a high-friction configuration flow: diary studies + intercept interviews surfaced the top 5 drop-off points; shipped in 6 months with a 3-person eng team."
- **ATS keywords carried:** UX design, usability testing, mixed methods, diary study, user interviews, task-completion time, cross-functional collaboration, product design, prototyping, Figma.
- **Accuracy notes:** ⚠️ MAU count is PM-reported estimate — present as "~80,000" with appropriate hedging until verified from a primary analytics source. ✅ Usability n=24 is applicant-confirmed. Do NOT inflate "collaborated with 3 engineers" to "led an engineering team."
